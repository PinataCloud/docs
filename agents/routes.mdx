Routes expose services running inside your agent container to the internet. Use routes to make APIs, web apps, and other services publicly accessible.

## Add a Route

1. Open your agent's detail page
2. Click **Routes** in the sidebar
3. Click **ADD**
4. Choose **Path Route** or **Custom Domain**

## Path Routes

Path routes forward traffic from a URL path to an internal port.

1. Click **ADD** → **Path Route**
2. Enter a **Path Prefix** (e.g., `/app`)
3. Enter the **Port** your service runs on
4. Choose **Protected** or **Public**
5. Click **Save**

Your service is now available at `https://agents.pinata.cloud/v0/agents/{agentId}/app/...`

The path prefix is stripped before reaching your service. A request to `/app/api/users` arrives at your service as `/api/users`.

### Path Format

- Must start with `/`
- Letters, numbers, hyphens, underscores only
- Cannot use reserved paths (`/v0`, `/chat`, `/config`)

## Custom Domains

<Note>
  Custom Domains is currently in beta.
</Note>

Custom domains route traffic from a dedicated hostname to your service. Unlike path routes, the full request path is forwarded as-is.

### Pinata Subdomain

Get a random subdomain on `*.apps.pinata.cloud`:

1. Click **ADD** → **Custom Domain** → **Pinata Subdomain**
2. Enter the **Port**
3. Choose **Protected** or **Public**
4. Click **Save**

You'll get a URL like `https://swift-fox-347.apps.pinata.cloud`.

### Your Own Domain

Use your own domain (e.g., `app.example.com`):

1. Click **ADD** → **Custom Domain** → **Bring Your Own**
2. Enter your **Domain**
3. Enter the **Port**
4. Choose **Protected** or **Public**
5. Click **Save**

Then configure DNS:

1. Add the **TXT record** shown to verify ownership
2. Add the **CNAME record** to point to the Pinata proxy

Status progression: `pending` → `pending_ssl` → `active`

## Protected vs Public

| Mode | Behavior |
| ---- | -------- |
| **Protected** | Requests require a gateway token |
| **Public** | Open to the internet |

<Warning>
  Public routes expose your service without authentication. Only use for services that handle their own auth.
</Warning>

## Constraints

| Limit | Value |
| ----- | ----- |
| Path routes per agent | 10 |
| Custom domains per agent | 5 |
| Port range | 1025–65535 |
| Reserved port | 18789 (gateway) |
