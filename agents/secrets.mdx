Secrets are encrypted credentials available as environment variables in your agents. Manage them at [agents.pinata.cloud/secrets](https://agents.pinata.cloud/secrets).

<Warning>
  Secret values cannot be viewed after saving. Store credentials securely before adding.
</Warning>

## Connect an LLM Provider

Every agent needs an LLM provider. Go to [agents.pinata.cloud/secrets](https://agents.pinata.cloud/secrets) and you'll see provider cards at the top:

1. Click **Connect** on your preferred provider (Anthropic, OpenAI, or OpenRouter)
2. Paste your API key
3. Click **Save**

The card will show "Connected" when configured.

### OpenAI OAuth Option

OpenAI supports OAuth connection through a Codex subscription:

1. Install the [Pinata CLI](/tools/cli/overview)
2. Run `pinata agents auth openai --oauth`
3. Complete the browser flow

This stores `OPENAI_OAUTH_TOKEN` in your vault automatically.

## Add a Secret

For API keys, tokens, and other credentials:

1. Click **Add Secret** (or use **⌘K** → "Add Secret")
2. Enter a **Name** (this becomes the environment variable name)
3. Enter the **Value**
4. Click **Save**

Your secret is encrypted and stored. It will be available as an environment variable in any agent you attach it to.

## Add a Variable

For non-sensitive configuration values:

1. Click **Add Variable**
2. Enter a **Name** and **Value**
3. Click **Save**

Variables work the same as secrets but aren't encrypted.

## Import from .env

Bulk import secrets from an existing `.env` file:

1. Click **Import .env file**
2. Select your file
3. Review the parsed values
4. Click **Import**

## Attach Secrets to an Agent

**During creation:** Add secrets in Step 3 (Connect) under "Variables and Secrets".

**After deployment:** Open your agent → **Secrets** → Add from your vault.

<Note>
  New secrets require an agent restart to take effect.
</Note>

## Update or Delete

Click the **...** menu on any secret:

- **Update** — Change the value (requires agent restart)
- **Delete** — Remove from vault (must detach from agents first)

## Using the CLI

```bash
# List secrets
pinata agents secrets list

# Add a secret
pinata agents secrets add <name> <value>

# Delete a secret
pinata agents secrets delete <name>
```

## Security

- Encrypted at rest with AES-GCM
- Unique derived key per user
- Values never returned by API
- Injected as environment variables, not written to disk
